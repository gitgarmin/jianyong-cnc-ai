# 安全审查 Skill

> 通用安全检查（SQL 注入、XSS、CSRF、密钥管理等）见全局规则 `~/.claude/rules/common/security.md`。
> 本 Skill 只关注**本项目特有的安全关注点**。

## When to Activate

仅在以下路径相关场景激活，其他目录下不加载：

- 当前编辑的文件在 `backend/app/core/security.py`
- 当前编辑的文件在 `backend/app/api/` 目录下且涉及认证/输入处理
- 当前编辑的文件在 `frontend/src/` 且涉及用户输入、文件上传
- 用户要求做安全审查或安全加固

## 本项目特有安全关注点

### 1. G 代码安全引擎（ADR-4）

安全校验引擎**独立于 AI，确定性执行**。审查时关注：

- [ ] 新增校验规则是否有对应的 positive + negative 测试
- [ ] 规则是否覆盖了所有危险模式（碰撞、断刀、超速）
- [ ] AI 生成的 G 代码**绝不绕过**安全引擎校验
- [ ] 规则严重级别（A/B/C）是否合理——A 级必须阻断，不能降级为警告

### 2. 图纸上传安全

用户上传 DXF/STEP/PDF 图纸，审查时关注：

- [ ] 文件类型白名单校验（只允许 `.dxf`, `.step`, `.stp`, `.pdf`）
- [ ] 文件大小限制（建议 ≤ 20MB）
- [ ] 文件名清理（防止路径遍历 `../../` 和特殊字符）
- [ ] 上传目录不在 web 可访问路径下
- [ ] 文件存储使用 UUID 重命名（不暴露原始文件名）

### 3. DeepSeek API Key 管理

- [ ] API Key 只从环境变量读取，绝不硬编码
- [ ] `.env` 在 `.gitignore` 中（已配置）
- [ ] API 调用失败时，错误响应不暴露 Key 或内部 URL
- [ ] 支持 `local` provider 降级（离线场景）

### 4. AI 输出安全

AI 可能生成不安全的内容，审查时关注：

- [ ] AI 建议的 G 代码必须经过安全引擎二次校验
- [ ] AI 回答中的技术参数（切削速度、进给量）有合理范围限制
- [ ] AI 不能直接执行写操作（只能建议，用户确认后才执行）
- [ ] RAG 检索结果不泄露其他用户的加工数据

### 5. CNC 加工物理安全

本项目的"安全"不仅是信息安全，还包括人身和设备安全：

- [ ] 安全校验规则的优先级：A 级（阻断）> B 级（警告）> C 级（提醒），不可降级
- [ ] 缺少程序结束指令（M02/M30）的代码必须标记
- [ ] 在运动指令前必须声明坐标系
- [ ] 在切削运动前必须启动主轴

## 本项目安全相关文件

| 文件 | 安全职责 |
|------|---------|
| `app/core/security.py` | JWT 创建/验证、密码哈希 |
| `app/core/config.py` | 环境变量读取（密钥、数据库 URL） |
| `app/services/gcode_engine.py` | G 代码安全校验引擎（28 条规则） |
| `app/services/ai_provider.py` | API Key 管理、Provider 降级 |
| `app/api/drawings.py` | 图纸上传、文件校验 |
| `.env` | 敏感配置（绝不提交到 git） |
