# 安全审查 Skill

## When to Activate

- 涉及认证或授权代码（JWT、密码、权限）
- 处理用户输入（表单、API 请求参数、文件上传）
- 数据库查询（防止 SQL 注入）
- 文件系统操作（防止路径遍历）
- 修改 `backend/app/core/security.py`
- 用户要求做安全审查或安全加固

## 审查检查清单

按顺序逐项检查：

### 1. 认证与授权

- [ ] JWT token 是否有过期时间
- [ ] 密码是否使用 bcrypt 哈希（不用明文或 MD5）
- [ ] 受保护端点是否验证 token
- [ ] 角色权限检查是否到位

### 2. 输入验证

- [ ] 所有 API 输入是否使用 Pydantic Schema 验证
- [ ] 文件上传是否检查文件类型和大小
- [ ] 字符串字段是否有 `max_length` 限制
- [ ] 数值字段是否有合理范围

### 3. 数据库安全

- [ ] 是否使用 SQLAlchemy ORM（自动防 SQL 注入）
- [ ] 是否有原始 SQL 拼接（危险，需改为参数化查询）
- [ ] 数据库连接是否使用环境变量（不硬编码）

### 4. 错误信息

- [ ] API 错误响应是否暴露堆栈信息（不应暴露）
- [ ] API 错误响应是否暴露内部路径或数据库结构
- [ ] 日志中是否记录了敏感信息（密码、token）

### 5. 文件操作

- [ ] 文件路径是否经过验证（防止路径遍历 `../../etc/passwd`）
- [ ] 上传文件是否存储在安全目录（不在 web 可访问路径）
- [ ] 文件名是否经过清理（防止特殊字符）

## 本项目安全相关文件

| 文件 | 安全职责 |
|------|---------|
| `app/core/security.py` | JWT 创建/验证、密码哈希 |
| `app/core/config.py` | 环境变量读取（密钥、数据库 URL） |
| `app/api/*.py` | 输入验证、错误处理 |
| `app/services/ai_provider.py` | API Key 管理 |
| `.env` | 敏感配置（绝不提交到 git） |

## 常见安全反模式

- 硬编码 API Key 或密码
- `except Exception: pass` 静默吞掉错误
- `print()` 输出敏感信息
- 用户输入直接拼接到 SQL 或 shell 命令
- 文件上传不限制类型和大小
