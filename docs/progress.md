## 优先级说明

| 级别 | 含义 |
|------|------|
| **P0** | 当前阶段必须完成，否则当前 milestone 无法交付 |
| **P1** | 当前阶段应该完成，但不阻塞 milestone 的核心目标 |
| **P2** | 下阶段再做，MVP 后可延期 |

## 并行标记

| 标记 | 含义 |
|------|------|
| 🔀 | 可与其他 🔀 标记的模块并行开发 |
| 🔒 | 必须等待「依赖」列中的模块完成后才能开始 |
| — | 已完成，不再适用 |

## 开发阶段总览

```
Phase 0 — 对话问答 MVP（2-3周）
  ├─ 🟦 后端: ORM → G代码引擎完善 → RAG检索 → /api/chat/send（对接前端）
  └─ 🟦 前端: 对话问答消息收发 → 对接API

Phase 1 — G代码生成核心链路（6-8周）
  ├─ 后端: ORM（复用）→ G代码引擎 → G代码生成API → 后处理模板 → 图纸解析API
  └─ 前端: 步骤1（上传+质检+下拉）→ 步骤2（解析展示+置信度+编辑）→ 步骤3（代码展示+内联编辑+安全确认）→ 步骤4（首检+刀具）

Phase 2 — 完整流程 + 增强（4-6周）
  ├─ 后端: 双图纸→加工余量计算→B类警告→多机床后处理→首检API→刀具追踪API
  ├─ 前端: 双图纸流程→首检清单→刀具管理→我的Tab→离线策略
  └─ 基础设施: CI/CD → PRD归档
```
